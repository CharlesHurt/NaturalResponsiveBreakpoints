Allthough there are codified media-query breakpoints in Bootstrap, there is another (sometimes better) way to address responsiveness.

There is no reason why natural break points cannot be implemented using media queries. For example, lets say that you have a design which takens up 3 columns on a desktop. If a column contains more than one word in the title, then a media query can be designed so that stacking occurs just before the page width contracts enough to cause the two header words to split.

There are a couple of contitions that must be met in order for this to work. First, the font size must be in units that can be depended on when calculating breakpoints. For simplicity, I will use "px". For single to conform to the desired width. Here is an example
