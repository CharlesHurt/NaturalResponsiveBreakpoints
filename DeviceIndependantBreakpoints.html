Device-Independent Responsive Breakpoints

Allthough there are codified media breakpoints in Bootstrap, there is another (sometimes better) way to address responsiveness.

"Natural" break points can be implemented using media queries. For example, lets say that you have a design which takens up 3 columns on a desktop. If a column contains more than one word in the title, then a media query can be implemented so that stacking occurs just before the page width contracts enough to cause header words to split.

There are a couple of contitions that must be met in order for this to work. First, the font size must be in units that can be depended on when calculating breakpoints. For simplicity, I will use "px". For a header with a single work, add a placeholder word to artificially create to the desired width.

There are two examples below. The first simply wraps text when forced to. The second example is controlled by a media query that forces the column to stack vertically, instead of word wrapping the title. This is done using "display:block" at smaller window widths.
